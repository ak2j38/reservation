# 도메인 설계

## 필수 도메인
- User
  - 간단하게 구현, 인증과 권한 관리에 집중(예약 서비스는 작업 가능한 유저만 수행)
  - 유저의 ID는 UUID로 해야하나?
  - 토큰에서는 UUID로 해당 유저를 식별할 수 있어야한다
  - 하나의 유저는 각 콘서트의 좌석을 여러 개 소유 가능
- Seat
  - 소유권 필요, 좌석 상태 관리, 등급, 가격
  - 어느 콘서트의 좌석인지 관리 필요
  - 하나의 좌석은 유저 한 명이 소유 가능(OneToMany)
- Point
  - 충전, 조회 
- Concert
  - 공연 정보 관리(날짜, 좌석, 이름)
  - 여러 개의 좌석 관리(OneToMany)
- Payment
  - 결제 처리
- Reservation?, WaitingQueue?
